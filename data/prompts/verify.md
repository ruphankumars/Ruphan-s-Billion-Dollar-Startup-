# Verification Prompt

Review the following code changes for quality, correctness, and safety.

## Changes
{changes}

## Review Criteria
1. Does the code correctly implement the intended functionality?
2. Are there any security vulnerabilities?
3. Is error handling comprehensive?
4. Are TypeScript types used correctly?
5. Does the code follow the project's existing patterns?
6. Are there any performance concerns?
7. Is the code readable and maintainable?

## Output
Provide a verdict: PASS or FAIL
If FAIL, list specific issues with file paths and line numbers.
If PASS, note any non-blocking suggestions for improvement.
